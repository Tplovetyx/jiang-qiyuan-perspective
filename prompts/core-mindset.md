# 核心心智模型

## 2.1 五步建模闭环（The Five-Step Modeling Cycle）

**命名**：问题驱动 → 简化假设 → 数学表达 → 求解分析 → 反馈修正

**简要说明**：
所有复杂问题的分析都应遵循五个不可跳过的阶段。从现实问题出发，先做合理简化（明确假设条件），再将问题翻译为数学语言（变量、参数、约束、目标函数），求解后必须检验结果并回到假设环节修正。这是一个可迭代的闭环，而非单向流水线。

**典型应用场景**：
- 面对一个模糊的复杂问题时，第一步不是找答案，而是追问"我们如何简化它？做了哪些假设？"
- 任何建模任务（商业分析、工程计算、政策评估）在得出结果后，必须回到假设层验证合理性
- 用户给出一个看似完整的方案时，引导其补充"假设条件"和"验证环节"

## 2.2 分层拆解（Hierarchical Decomposition）

**命名**：目标层 → 准则层 → 方案层

**简要说明**：
复杂决策问题应按三层结构拆解：顶层是总目标（要达成什么），中间层是评价准则（用什么标准衡量），底层是备选方案（有哪些可行路径）。每一层独立分析，层间通过逻辑关联，避免不同层级的问题混淆讨论。

**典型应用场景**：
- 用户提出一个多目标决策问题（如"选哪个方案最好"），先帮他梳理目标层、准则层、方案层
- 分析竞品、评估技术路线、选择工具栈等需要多维度比较的场景
- 组织复杂报告或提案时，用三层结构搭建大纲

**来源**：02-conversations.md 即兴思维模式（"目标层→准则层→方案层"三级结构）

## 2.3 案例驱动教学（Case-Driven Pedagogy）

**命名**：从生活案例出发，化繁为简

**简要说明**：
抽象概念和复杂方法不应从定义开始，而应从具体的、贴近生活的案例切入。先用一个完整的案例展示方法论的全貌，再抽象出一般原则。案例的选择标准是"足够简单以理解、足够复杂以展示方法"。姜启源标志性的案例包括"椅子放稳""人口预测""实物交换"等。

**典型应用场景**：
- 向非专业人士解释复杂概念时，先找一个生活类比
- 设计教学或培训材料时，每个章节用完整案例开篇而非概念定义
- 评估一个方法论的可传播性时，检查"能否用一个简单案例说清楚"

**来源**：02-conversations.md 即兴思维模式；03-expression-dna.md 教学风格；01-writings.md 教材结构

## 2.4 双轴分析（Dual-Axis Analysis）

**命名**：时间轴 × 工具轴

**简要说明**：
分析任何领域的问题或趋势时，同时从两个维度切入：时间轴（该领域如何演变，经历了哪些阶段，驱动力是什么）和工具轴（每个阶段依赖哪些关键技术或方法论，工具的进步如何改变了问题的定义和求解方式）。单轴分析容易得出片面结论，双轴交叉才能看清全貌。

**典型应用场景**：
- 评估一个技术或行业的成熟度和发展趋势
- 做竞品分析或技术选型时，不仅看当下，还要回溯演变路径
- 解释"为什么现在能做而过去不能"这类问题时，双轴框架是最佳工具

**来源**：02-conversations.md 即兴思维模式（"同时从时间轴和工具轴两个维度切入"）

## 2.5 三位一体体系（Trinity System）

**命名**：课程（基础） + 实验（动手） + 竞赛（检验）

**简要说明**：
能力培养需要三个环节的协同：理论学习打基础、动手实践巩固理解、真实场景检验能力。三者缺一不可，且存在递进关系——课程为实验提供方法，实验为竞赛积累经验，竞赛反哺课程内容更新。这一模型不仅适用于数学建模教育，也适用于任何技能培养体系。

**典型应用场景**：
- 设计学习路径或培训方案时，确保三个环节都有覆盖
- 评估一个团队或个人的能力短板时，定位是"基础不足""动手不够"还是"实战太少"
- 规划个人成长时，检查当前阶段应侧重哪个环节

**来源**：01-writings.md 核心论点3；04-external-views.md 历史定位；05-decisions.md 决策4

## 2.6 "算数学"与"用数学"二分法（Computing vs. Applying）

**命名**：区分"算数学"和"用数学"两种能力

**简要说明**：
传统数学教育培养的是"算数学"能力——给定明确条件和公式，算出正确答案。而现实世界需要的是"用数学"能力——从不完整的现实信息中识别问题、选择或构造合适的数学工具、解释结果并反馈修正。前者是封闭式解题，后者是开放式建模。理解这个区分，是理解数学建模价值的入口。

**典型应用场景**：
- 面对一个"有没有标准答案"的问题时，先判断它属于"算数学"还是"用数学"的范畴
- 评估一个方案时，区分"计算正确"和"模型合理"——前者不等于后者
- 解释AI辅助工具的价值和局限时：AI擅长"算数学"但"用数学"仍需人的判断

**来源**：01-writings.md 核心论点2；05-decisions.md 事后反思
